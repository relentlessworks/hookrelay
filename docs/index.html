Quick Start

# Build the single static binary
make build

# Run with zero config — starts on :8080
./hookrelay

# Create a webhook endpoint
curl -X POST http://localhost:8080/api/endpoints \
  -H "Authorization: Bearer <token>" \
  -d "target_url=https://your-service.com/webhook"

# Response (plain text, one line per record)
handle=hook_a1b2c target_url=https://your-service.com/webhook

# Send a webhook — HookRelay forwards it to your target
curl -X POST http://localhost:8080/hook/hook_a1b2c -d '{"event":"test"}'
handle=del_x1y2z status=200 delivered=true

# Inspect deliveries
curl http://localhost:8080/api/endpoints/hook_a1b2c/deliveries \
  -H "Authorization: Bearer <token>"

Principles

🤖

Agent is the interface

No UI, no SDK. The AI agent is the user. The API is the product.

📝

Plain text by default

One labeled, grepable line per record. Token-cheap, survives context truncation. JSON on demand.

💡

Instructive errors

Every 4xx includes a hint telling the agent what to do next. Self-correcting without a schema.

📖

Self-documenting

GET /help returns a one-page operating manual the agent loads at runtime.

🔐

Simple auth

OTP via email → long-lived bearer token. No complex OAuth flows.

📦

Single static binary

Go, zero external dependencies, CGO_ENABLED=0. Deploys as one file.

⚙️

Zero config defaults

Runs out of the box. Config layered: defaults < file < env < flags.

🏢

Multi-tenant ready

Workspaces isolate endpoints and deliveries per tenant. Plans, limits, audit logs built in.

API Reference

Authentication

POST /auth/request   email=<email>&workspace=<handle>  → OTP code
POST /auth/verify    email=<email>&code=<code>          → Bearer token

Endpoints (requires Bearer token)

MethodPathDescription
POST/api/endpointsCreate a webhook endpoint
GET/api/endpointsList all endpoints
GET/api/endpoints/<handle>Endpoint details
DELETE/api/endpoints/<handle>Delete an endpoint
GET/api/endpoints/<handle>/deliveriesList deliveries for an endpoint
GET/api/deliveries/<handle>Delivery details
GET/api/workspaceWorkspace info

Webhook Receiver (public, no auth)

POST /hook/<handle>  → Forwards body to target_url, returns delivery handle + status
# Any HTTP method accepted (GET, POST, PUT, DELETE, PATCH)

Response Formats

# Plain text (default)
handle=hook_a1b2c target_url=https://example.com/webhook deliveries=3

# JSON (add Accept: application/json or ?format=json)
{"handle":"hook_a1b2c","target_url":"https://example.com/webhook","deliveries":3}
